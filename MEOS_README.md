# 小鸣同学 - 使用说明

## 📋 系统概述

小鸣同学是一个智能客服系统，帮助用户快速获得问题解答。系统支持知识库管理功能。

## 🎯 核心功能

### 1. AI 对话功能
- **文本输入**：直接输入问题，AI会基于知识库提供解答
- **图片上传**：支持上传 PNG/JPG 图片（≤10MB），AI可以理解图片内容
- **语音输入**：移动端支持语音转文本（演示功能）

### 2. 知识库管理
- **密码保护**：管理密码为 `sunshifu1688`
- **文档上传**：支持 PDF、DOC、DOCX、MD、TXT 格式（≤50MB）
- **文档管理**：可以查看、删除、批量删除文档
- **持久化存储**：文档存储在 Supabase Storage，元数据存储在 KV Store

### 3. 后端架构
- **Supabase Storage**：用于存储知识库文档
- **KV Store**：用于存储文档元数据
- **豆包大模型**：提供 AI 对话能力
- **API 接口**：
  - `POST /knowledge/upload` - 上传文档
  - `GET /knowledge/list` - 获取文档列表
  - `DELETE /knowledge/delete/:id` - 删除文档
  - `POST /knowledge/batch-delete` - 批量删除文档
  - `POST /chat/send` - 发送消息到 AI

## 🔧 技术栈

- **前端**：React + TypeScript + Tailwind CSS
- **后端**：Supabase Edge Functions (Hono)
- **数据库**：Supabase KV Store
- **存储**：Supabase Storage
- **AI**：豆包大模型 (Doubao-Seed-1.8)

## 🚀 使用流程

### 用户对话流程
1. 在聊天框中输入问题或上传图片
2. 系统将消息发送到后端
3. 后端调用豆包 API 处理请求
4. AI 基于知识库生成回复
5. 用户收到 AI 回复（响应时间 <3秒）

### 知识库管理流程
1. 点击"知识库管理"按钮
2. 输入管理密码：`sunshifu1688`
3. 上传文档到知识库
4. 文档会被存储到 Supabase Storage
5. AI 会基于这些文档回答用户问题

## ⚙️ 环境变量

系统已配置以下环境变量：
- `DOUBAO_API_KEY` - 豆包 API 密钥
- `DOUBAO_MODEL_ID` - 豆包模型 ID
- `SUPABASE_URL` - Supabase 项目 URL
- `SUPABASE_ANON_KEY` - Supabase 匿名密钥
- `SUPABASE_SERVICE_ROLE_KEY` - Supabase 服务角色密钥

## 💡 使用建议

1. **上传高质量文档**：上传详细的操作手册、FAQ 文档，AI 回答会更准确
2. **定期更新知识库**：保持知识库内容最新，提高问题解决率
3. **清晰提问**：用户提问越清晰，AI 回答越准确
4. **图片辅助**：遇到现场问题时，配合图片提问效果更好

1. 检查网络连接
2. 查看浏览器控制台错误日志
3. 确认后端服务是否正常运行
4. 验证环境变量是否正确配置
